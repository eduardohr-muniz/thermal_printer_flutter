## What this is

`thermal_printer_flutter` is a federated Flutter plugin for ESC/POS thermal printing over **USB**, **Bluetooth (RFCOMM/BLE)**, and **Network (raw TCP/IP)**. Targets Android, iOS, macOS, Windows, Linux, and Web. ESC/POS command bytes are generated by the re-exported `esc_pos_utils_plus` package — this plugin only handles transport.

## Commands

```bash
flutter pub get                          # install deps (run in repo root AND in example/)
flutter analyze                          # lint (CI gate)
flutter test                             # all tests
flutter test --coverage                  # with lcov → coverage/lcov.info
flutter test test/printer_test.dart      # single test file
flutter test --name "substring"          # single test by name

# Native builds (run from example/ — the plugin itself has no runnable app)
cd example && flutter build apk --debug          # Android
cd example && flutter build ios --no-codesign --debug
cd example && flutter build macos --debug
cd example && flutter build windows --debug
cd example && flutter build linux --debug
```

CI (`.github/workflows/ci.yml`) runs analyze + test on Ubuntu and builds the example app on all five native platforms. `flutter analyze` must be clean — `avoid_print` is enabled (use `debugPrint`/`log`), and `deprecated_member_use` is downgraded to ignore.

## Architecture

The layering follows Flutter's federated-plugin pattern, then **fans out by transport, not by OS**:

1. **`ThermalPrinterFlutter`** (`lib/thermal_printer_flutter.dart`) — public facade. Re-exports `esc_pos_utils_plus`, the models, and enums, so consumers import only this one file. Owns the **serial print queue** (`_enqueuePrint`): every `printBytes` chains onto the previous one so only one RAW job hits the spooler at a time. This exists to stop the duplicated/garbled output that happens when callers loop `printBytes`.

2. **`ThermalPrinterFlutterPlatform`** (`*_platform_interface.dart`) — the `PlatformInterface` abstract contract. Default instance is `MethodChannelThermalPrinterFlutter`.

3. **`MethodChannelThermalPrinterFlutter`** (`*_method_channel.dart`) — does **not** talk to a single native channel for everything. It dispatches each call by `PrinterType` to one of three Dart repositories. Repositories are constructor-injected (default to real impls) — that injection seam is how the tests run without native code.

4. **Repositories** (`lib/src/repositories/`), all implementing `PrinterRepository`:
   - `UsbPrinterRepository` & `BluetoothPrinterRepository` — thin Dart wrappers over `MethodChannel('thermal_printer_flutter')`; the real work is in native code.
   - `NetworkPrinterRepository` / `network_printer.dart` — **pure Dart** (`dart:io` Socket), no native channel. Holds a pooled socket; `dispose()` closes it.

5. **Native side** — all native code answers the single `thermal_printer_flutter` method channel:
   - Android (`android/.../ThermalPrinterFlutterPlugin.kt`) — Bluetooth + Network.
   - iOS (`ios/Classes/`, Swift + ObjC shim) — Bluetooth + Network.
   - macOS (`macos/Classes/`, Swift + `CupsPrinter.m`) — USB goes through **CUPS/the system print queue**, not raw USB; Bluetooth + Network too.
   - Windows (`windows/*.cpp`) — USB via the print spooler + Bluetooth via **RFCOMM/Winsock**.
   - Linux (`linux/*.cc`) — Network only.
   - Web (`thermal_printer_flutter_web.dart`) — experimental network only.

### Cross-cutting behaviors worth knowing before editing

- **`copies` is resolved in Dart, not the driver.** `printBytes(copies: n)` repeats the byte payload `n` times into one job. On Windows the native driver copy-count is forced to 1 so this Dart count is the single source of truth. Don't reintroduce driver-level copies.
- **`getPrinterStatus` only returns real data for USB** on Windows (spooler) and macOS (CUPS); every other type/platform returns `PrinterStatus.unknown`. The method-channel layer short-circuits non-USB before even calling native.
- **USB is connectionless** — `connect()` is a no-op returning `true`, and `isConnected` is always `true` for USB. Use `getPrinterStatus` for real health.
- **Bluetooth holds one active connection.** `disconnect()` closes the active one regardless of which `Printer` is passed.
- **Bluetooth address keys differ per OS:** native sends `bleAddress` (iOS/macOS) or `address` (Android); the repository accepts both. Preserve that fallback or Android `connect()` breaks with "MAC address is required".
- **Network discovery** flags any host with an open 9100/515/631 port; `requireConfirmation: true` adds an ESC/POS probe on 9100 to cut false positives.

## Models

`lib/src/models/printer.dart` (`Printer` — carries `type`, `name`, `bleAddress`, `usbAddress`, `ip`, `port`), `printer_status.dart` (`PrinterStatus` with `fromMap`), and `lib/src/enums/printer_type.dart` (`usb` / `bluetooth` / `network`). `lib/src/services/screenshot.dart` renders a widget to a monochrome (optionally Floyd–Steinberg dithered) `img.Image` for `generator.imageRaster`.

## Git

**Não faça `git commit` nem `git push` a menos que o usuário peça explicitamente.** Faça as alterações no working tree e deixe o commit/push para o usuário, salvo instrução direta em contrário.

## Conventions

- Doc comments and inline rationale are written in **Portuguese**; match that when editing existing files.
- Tests use **mocktail** and the repository injection seam — there is no native code in the test path. Add coverage by injecting fake repositories, not by mocking method channels directly where avoidable.
